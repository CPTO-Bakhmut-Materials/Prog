Читання `diff` (різницi) — це базова навичка для програміста, особливо при роботі з Git. Вивід команди `diff` показує, які рядки були змінені, додані або видалені між двома версіями файлу.

Найпоширенішим форматом зараз є **Unified Format** (Уніфікований формат), який використовують Git та GitHub. Розглянемо його в першу чергу.

-----

### 1\. Уніфікований формат (Unified Diff)

Цей формат ви бачите, коли вводите `git diff` або `diff -u файл1 файл2`.

Приклад виводу:

```diff
--- a/script.py
+++ b/script.py
@@ -1,4 +1,4 @@
 def greet(name):
-    print("Hello " + name)
+    print(f"Hello, {name}!")
     return True
```

#### Як це розшифрувати:

**1. Заголовок (Файли):**

  * `--- a/script.py`: "Мінус" позначає **старий** (оригінальний) файл.
  * `+++ b/script.py`: "Плюс" позначає **новий** (змінений) файл.

**2. Заголовок блоку змін (Hunk Header):**

  * `@@ -1,4 +1,4 @@`: Це координати змін.
      * `-1,4`: У старому файлі (`-`) цей шматок починається з рядка **1** і показує **4** рядки.
      * `+1,4`: У новому файлі (`+`) цей шматок починається з рядка **1** і показує **4** рядки.

**3. Самі зміни (Тіло):**
Кожен рядок починається з одного символу:

  * `     ` (Пробіл): Рядок **не змінювався**. Це контекст, щоб ви розуміли, де знаходиться код.
  * `-` (Мінус): Цей рядок був **видалений** із старої версії. (Зазвичай підсвічується червоним).
  * `+` (Плюс): Цей рядок був **доданий** у нову версію. (Зазвичай підсвічується зеленим).

> **Суть зміни в прикладі:** Рядок з конкатенацією (`+ name`) було замінено на f-string (`f"Hello...`), але для `diff` це виглядає як "видалення старого рядка і додавання нового".

-----

### 2\. Звичайний формат (Normal Diff)

Це формат за замовчуванням для стандартної утиліти `diff` (без прапорця `-u`). Він більш компактний, але менш читабельний для людей.

Приклад виводу для того ж файлу:

```text
2c2
<     print("Hello " + name)
---
>     print(f"Hello, {name}!")
```

#### Як це розшифрувати:

**1. Координати зміни (`2c2`):**
Літери вказують на дію, яку треба виконати над першим файлом, щоб він став як другий:

  * `a` (Add/Додати): Наприклад `1a2` — після рядка 1 додати рядок.
  * `d` (Delete/Видалити): Наприклад `3d2` — видалити рядок 3.
  * `c` (Change/Змінити): У прикладі `2c2` означає: рядок **2** у першому файлі треба **змінити**, щоб він відповідав рядку **2** у другому файлі.

**2. Зміст:**

  * `<`: Рядки з першого (старого) файлу.
  * `>`: Рядки з другого (нового) файлу.
  * `---`: Розділювач між старою та новою версіями при зміні (`c`).

-----

### 3\. Особливі випадки у Git

У Git ви можете побачити ще метадані на початку:

```diff
diff --git a/index.html b/index.html
index 83c4..29a1 100644
```

  * `index 83c4..29a1`: Це хеші (SHA-1) версій файлу "до" і "після".
  * `100644`: Це права доступу до файлу (mode). `644` — звичайний файл, `755` — виконуваний скрипт.

### Шпаргалка для швидкого читання

| Символ | Значення | Колір (зазвичай) | Дія |
| :--- | :--- | :--- | :--- |
| `---` | Старий файл | - | Дивись заголовок |
| `+++` | Новий файл | - | Дивись заголовок |
| `@@` | Початок блоку | Фіолетовий/Блакитний | Показує номери рядків |
| `-` | Цього більше немає | **Червоний** | Було видалено |
| `+` | Це нове | **Зелений** | Було додано |
| `     ` | Контекст | Сірий/Білий | Без змін (для орієнтування) |
